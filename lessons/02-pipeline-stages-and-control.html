Principle of Locality

Why programs don't access memory randomly — and why that's the key to fast computers.

👨‍🍳

The Chef Analogy

A chef prepares a complex menu. Instead of running to the pantry every time, they keep frequently used ingredients nearby — salt, oil, spices. Not everything can fit on the counter (capacity limit), but smart placement means less walking and faster cooking (performance boost).


Your CPU works the same way. Instead of accessing slow main memory every time, it keeps recently used data in fast cache memory.

⏱
Temporal Locality
Locality in Time

If data at address X is accessed, it will likely be accessed again soon. Think: loop counters, variables used inside loops.

// Example: loop counter accessed repeatedly
for (i = 0; i < 1000; i++) {
  sum += arr[i]; // 'sum' → temporal locality
}

Spatial Locality
Locality in Space

If address X is accessed, nearby addresses will likely be accessed soon. Think: sequential array access, instructions.

// Array elements accessed sequentially

// LEVEL 02 — MEMORY HIERARCHY

The Memory Pyramid

Multiple levels of memory with different speeds, sizes, and costs. Hover each level for details.

← FASTER, SMALLER, PRICIER  |  SLOWER, BIGGER, CHEAPER →
Data is copied between only two adjacent levels at a time. As we move away from the processor, access time increases progressively.
STANDARD MEMORY PROCESS
💾
1. Store everything on Disk
Magnetic disk — huge capacity, very slow
↑ copy recently accessed items ↑
🧠
2. Copy to DRAM (Main Memory)
Medium speed — much larger than cache
↑ copy more recently accessed items ↑
⚡
3. Cache (SRAM, attached to CPU)
Fastest — tiny but blazing fast
🎯 Goal: Whenever possible, keep references in faster memory — with minimal cost.
KEY TERMINOLOGY
✅ Cache HIT
Data found in upper level. Access is satisfied immediately.
Hit Rate = Hits / Total Accesses
❌ Cache MISS
Data absent — block copied from lower level. Pipeline stalls.
Miss Rate = 1 − Hit Rate
⏰ Miss Penalty
Time to copy block from lower level. Can be 100s of cycles for main memory.

SRAM vs DRAM vs Flash vs Disk

Four very different memory technologies — each with unique tradeoffs. Click to compare.

⚡
SRAM
Static Random Access Memory
Speed0.5–2.5 ns
Cost/GiB$500–$1000
CapacitySmall (KB–MB)
Used for: Cache memory (L1, L2, L3)
Fixed access time. No refresh needed. Minimal standby power. Retains data as long as powered.
🧠
DRAM
Dynamic Random Access Memory
Speed50–70 ns
Cost/GiB$10–$20
CapacityLarge (GB)
Used for: Main memory (RAM)
Data stored as charge in capacitor. Single transistor per bit → denser. Must be periodically refreshed (charge leaks).
📱
Flash
EEPROM-based Non-volatile
Speed5,000–50,000 ns
Cost/GiB$0.75–$1.00
CapacityLarge (GB–TB)
Used for: Mobile devices (SSD, phones)
Non-volatile — retains data without power. Between DRAM and disk in cost and speed.
💽
Magnetic Disk
Mechanical Storage
Speed5M–20M ns
Cost/GiB$0.05–$0.10
CapacityHuge (TB)
Used for: Servers, PCs (HDD)
Slowest non-volatile. Largest and cheapest storage. Mechanical seek time dominates.

CPI · AMAT · Miss Penalty

The math that shows why cache misses can make your CPU 2.72× slower. Interactive solver with the lecture examples.

AVERAGE MEMORY ACCESS TIME (AMAT)
AMAT = Hit Time + Miss Rate × Miss Penalty
Example: CPU with 1ns clock, hit time = 1 cycle, miss penalty = 20 cycles, miss rate = 5%
AMAT = 1 + 0.05 × 20 = 2 ns = 2 cycles per instruction
CPI WITH MEMORY STALLS CALCULATOR
CPU time = (Execution Cycles + Memory Stall Cycles) × Clock Cycle Time
Memory stalls = (Accesses/Instruction) × Miss Rate × Miss Penalty

Write Policies: Write-Through vs Write-Back

Write-Through
Every write updates both cache AND memory.
✅ Simple, always consistent
❌ Slower — every write hits memory

Example: Base CPI=1, 10% stores, memory=100 cycles
Effective CPI = 1 + 0.1×100 = 11
Write Buffer: A queue that holds data while waiting to write to memory — reduces stalls without compromising consistency.
Write-Back
Write only to cache block. Write to memory only when block is evicted.
✅ Faster — writes stay in cache
❌ More complex, uses "dirty bit"

Uses a dirty bit per cache block to track modifications.
Best when: Processors generate writes faster than main memory can handle them (common in modern CPUs).

Direct · Set · Fully Associative

Compare all three mapping strategies with the exact lecture example. Access sequence: 0, 8, 0, 6, 8.

Access sequence: 0 → 8 → 0 → 6 → 8 (4-block cache)
CONCEPT COMPARISON
Direct Mapped
Each block → exactly one cache location
Formula: addr mod #blocks
✅ Fast lookup (1 comparator)
❌ Conflict misses
n-Way Set Associative
Block → one of n slots in a set
Set = addr mod #sets
✅ Fewer conflicts
n comparators per set
Fully Associative
Block → any cache location
Search all entries simultaneously
✅ No conflict misses
❌ Expensive — comparator per entry
// LEVEL 09 — REPLACEMENT POLICY

LRU — Least Recently Used

When cache is full and a miss occurs, which block do we evict? Simulate LRU replacement interactively.

In LRU: the block that hasn't been accessed for the longest time gets replaced on a miss. The LRU state is tracked per block — "most recently used" stays, "least recently used" leaves.
LRU QUEUE (Left = Most Recent, Right = Victim)
IN DIRECT-MAPPED CACHE
When a miss occurs in direct-mapped cache, there is only one possible location — so replacement is automatic (no choice). LRU is relevant for set-associative and fully associative caches where multiple candidates exist. For 2-way set-associative with 2 blocks per set, we track which was used less recently.
// LEVEL 10 — ADVANCED ARCHITECTURE

Multilevel Cache

Modern CPUs use L1, L2, L3 caches. Simulate a memory access traveling through the hierarchy.

MULTILEVEL HIERARCHY SIMULATION
WHY MULTILEVEL CACHES?
The Problem: Fast CPUs need fast memory access, but main memory is 100× slower.

The Solution: Add an intermediate L2 cache on-chip.

When L1 misses: check L2 first. L2 hit → miss penalty = L2 access time (much less than RAM).

When both miss: access main memory (large penalty).

Result: Smaller average miss penalty → better AMAT.
MULTILEVEL AMAT FORMULA
AMAT = L1_hit_time
+ L1_miss_rate × (L2_hit_time
+ L2_miss_rate × Mem_access_time)

// Example values:
L1 hit = 1 cycle (SRAM)
L2 hit = 10 cycles (on-chip SRAM)
RAM = 200 cycles (DRAM)
L1 miss rate = 5%
L2 miss rate = 1%

AMAT = 1 + 0.05 × (10 + 0.01 × 200)
= 1 + 0.05 × 12
= 1.6 cycles
